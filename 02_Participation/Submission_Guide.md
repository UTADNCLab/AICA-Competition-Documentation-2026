# Submission Guide 

## Submission Link For Virtual Stage: **LINK**

This guide defines how to prepare and record a valid final submission video for AICA virtual-stage evaluation.

---

## 1) Purpose

Your submission must clearly demonstrate:

- Autonomous or manual-assisted execution of the delivery mission
- Correct setup and launch sequence
- Continuous run from start to finish
- Final score visibility in QLabs

---

## 2) Mandatory Submission Format

**Drive or YouTube link** to your final run video

 ---

## 3) Video Recording Rules

### A. Start from execution setup
Your recording should begin before mission execution starts, including:

- Opening environment/tools
- Running required startup or BAT flow
- Launching the scenario and executing the navigator for QCar2 and QDrone2.

### B. Single continuous run (no edits)
The final submission video should be **one continuous, uncut run**:

- No scene cuts
- No post-edit stitching
- No rerun fragments merged together
- No fast-forward sections

### C. No restart during official submitted run
For the submitted attempt:

- Do not stop and restart midway
- Do not replace failed parts with a second run
- Only the final full-stretch run should be submitted

---

## 4) Delivery Completion Requirement

To be considered complete:

- **All deliveries must be completed** in the submitted run
- Completion should be visible in simulation outputs

If all deliveries are not completed:

- The mission is considered **incomplete**
- Final scoring will reflect incomplete objective completion
- Points are awarded according to completed progress and official scoring logic

---

## 5) Timing and Scoring Visibility

Your video should clearly show:

- Mission start timing point
- Runtime progression during deliveries
- Final mission completion state
- Final score indicators visible in QLabs window.
This allows reviewers to evaluate delivery performance and completion quality consistently.

---

## 6) Recommended Video Flow

1. Run startup script / BAT file.
2. Start mission
3. Autonomous or manual-assisted execution (Qcar2 + Qdrone2 coordination)
4. Show all delivery completions
5. Show final time and score
6. End recording only after final results are visible

---

## 7) Reference Example

Use your attached sample video as a reference for:

- How delivery is completed 
- How points are visible in the QLabs window
- How to present clear execution flow for evaluation

---

## 8) Reviewer Expectation Checklist

Before submitting, verify:

- [ ] Video is a single continuous run
- [ ] Startup and execution steps are visible
- [ ] No edits/cuts/rerun patching
- [ ] All deliveries completed
- [ ] Final score and time visibility included
